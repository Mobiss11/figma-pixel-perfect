# 🎯 figma-pixel-perfect

**Скилл для Claude Code / Cursor / любого агента с Figma MCP: вёрстка по макету без отсебятины.**

> Ты не дизайнер. Ты фотокопир с линейкой.

## Проблема

Даёшь агенту ссылку на Figma — он подключается к MCP, что-то тянет… а потом **рисует иконки по памяти**, округляет 13px до 12px «для сетки», подставляет «похожий» шрифт и сдаёт работу со словами «готово, pixel perfect». Знакомо?

Причина: AI оптимизирует под «выглядит правдоподобно», а не под «совпадает с макетом». Без жёсткого протокола он будет творить.

## Решение

Скилл превращает вёрстку по Figma в проверяемый протокол:

| | Без скилла | Со скиллом |
|---|---|---|
| Иконки | рисует «по смыслу», берёт из lucide | **только скачанные из макета**, число ассетов сверяется с DOM |
| Размеры | «примерно так», округляет | дословно из design context, `p-[18px]` ≠ `p-4` |
| Проверка | «вроде похоже» | **численный аудит**: `getBoundingClientRect`/`getComputedStyle` против спеки + скриншот-дифф с локализацией зон |
| Проблемы | молча импровизирует | блок «⚠️ Проблемы по макету»: нет шрифта, не выгрузился ассет, нет hover-состояния — **спросит, а не придумает** |
| Токены | жжёт лимиты повторными вызовами MCP и разглядыванием картинок | один заход за данными → дистилляция в спек-файл → проверка текстом, правки пачкой |

## Как это работает

```
figma-ссылка
   │
   ├─ 1. Данные: get_design_context (главный) + metadata + screenshot + variables
   │      → дистилляция в _figma_spec.md (больше к MCP не ходим)
   ├─ 2. Ассеты: batch-выгрузка всех иконок/картинок из макета
   ├─ 3. Вёрстка: значения дословно, токены проекта где есть, ничего сверх макета
   ├─ 4. Численный аудит: measure_dom.js → дельты «спека 24px / рендер 28px»
   ├─ 5. Визуальная сверка: pixel_diff.py → % расхождения + heatmap + горячие зоны 4×4
   │      (антиалиасинг ≠ баг: значения, совпадающие со спекой, не трогаются)
   └─ 6. Отчёт: таблица дельт + «⚠️ Проблемы по макету»
```

Протокол собран на официальных рекомендациях [figma/mcp-server-guide](https://github.com/figma/mcp-server-guide) и полевых граблях pixel-perfect-воркфлоу 2026 (measurement-driven verification вместо наивного скриншот-диффа).

## Установка

**Claude Code (глобально):**

```bash
git clone https://github.com/Mobiss11/figma-pixel-perfect ~/.claude/skills/figma-pixel-perfect
```

**В конкретный проект:** положи папку в `.claude/skills/figma-pixel-perfect`.

**Cursor и другие:** подключи `SKILL.md` как rules/knowledge — протокол самодостаточен.

Требования: любой Figma MCP-сервер (официальный Dev Mode, remote или community), Python 3 + Pillow для `pixel_diff.py` (`pip install pillow`).

## Что внутри

```
figma-pixel-perfect/
├── SKILL.md                     # протокол: железные правила, шаги, анти-паттерны
├── scripts/
│   ├── measure_dom.js           # численный аудит DOM (rect + computed styles + счётчики иконок)
│   └── pixel_diff.py            # скриншот-дифф: %, heatmap, горячие зоны, retina-нормализация
├── references/
│   └── mcp-tools.md             # карта Figma MCP-серверов, приоритет хинтов, error recovery
├── templates/
│   └── figma-spec.md            # спек-файл: дистиллят макета для аудита без повторных вызовов MCP
└── tests/                       # pytest: структура скилла + смоук pixel_diff
```

## Фишки, которых нет в базовых figma-скиллах

- **Несколько MCP-серверов**: у тебя разные проекты под разными аккаунтами Figma? Скилл сам находит сервер с доступом к файлу дешёвым пробным вызовом.
- **Детектор выдуманных иконок**: N скачанных ассетов = N `<svg>/<img>` в рендере. Дубль или самодеятельность ловятся DOM-запросом, а не глазами.
- **Защита от «пиксельной охоты»**: лимит 3 итерации по визуальному диффу и правило «совпадает со спекой — не трогай», чтобы агент не ломал верные значения, гоняясь за шумом антиалиасинга.
- **Экономия лимитов**: спек-файл вместо повторных вызовов MCP, текстовые проверки вместо разглядывания скриншотов, правки пачкой.
