# Карта Figma MCP-серверов и их инструментов

У пользователя может быть подключено несколько Figma MCP одновременно (разные проекты/аккаунты). Имена серверов различаются, но инструменты узнаваемы по именам. Ищи через ToolSearch по слову `figma` или `design_context`, если инструменты отложенные.

## Официальный Figma MCP (Dev Mode / remote)

Ключевые инструменты (могут быть с любым префиксом `mcp__<server>__`):

| Инструмент | Что даёт | Заметки |
|---|---|---|
| `get_design_context` | Референс-код (React+Tailwind), скриншот, хинты: Code Connect, аннотации, токены | **Первый и главный вызов.** Официальная позиция Figma: metadata/screenshot — НЕ замена ему, только ориентация и валидация. На больших нодах ответ урезается — спускайся к дочерним нодам по id из metadata. Возвращённый код — референс, не финал: адаптируй под стек проекта. |
| `get_metadata` | XML-дерево слоёв: id, имя, тип, x/y/width/height | Дешёвый. Для ориентации (какую ноду брать), таблицы ожидаемых габаритов и размера вьюпорта. |
| `get_screenshot` | Рендер ноды картинкой | ЭТАЛОН для визуальной сверки. Сохраняй в файл сразу. |
| `get_variable_defs` | Переменные/токены: имена цветов, размеры | Полезно для читаемых имён в коде. |
| `download_assets` / `upload_assets` | Выгрузка ассетов (SVG/PNG) по node id | Есть не во всех вариантах сервера — тогда ассеты приходят ссылками в design context. |
| `whoami` | Текущий аккаунт | Быстрый способ понять, какой сервер под каким аккаунтом. |

## Community-серверы (Framelink / figma-developer-mcp и т.п.)

| Инструмент | Аналог |
|---|---|
| `get_figma_data` | metadata + context одним вызовом (JSON, упрощённый) |
| `download_figma_images` | download_assets |

Если доступен только community-сервер: значения бери из его JSON, скриншот эталона снимай через официальный сервер, а если его нет — попроси пользователя дать экспорт фрейма (PNG), без эталона самопроверка невозможна.

## Разбор ссылок

```
https://www.figma.com/design/AbC123/My-App?node-id=45-678&t=...
                          ^fileKey            ^nodeId "45:678" (дефис → двоеточие)
https://www.figma.com/file/AbC123/...   — старый формат, fileKey тот же
https://www.figma.com/proto/AbC123/...  — прототип; для вёрстки нужна design-нода,
                                          попроси ссылку на фрейм (Copy link to selection)
```

Без `node-id` ссылка указывает на файл целиком → получи metadata страницы, покажи список фреймов, спроси какой нужен.

## Выбор сервера при нескольких подключённых

1. Собери список серверов с figma-инструментами.
2. По очереди дешёвым вызовом (`get_metadata` на fileKey) проверь доступ.
3. 403/404/«file not found» → следующий сервер.
4. Никто не имеет доступа → репорт пользователю с именами проверенных серверов. НЕ верстай вслепую.

## Приоритет хинтов из design context (официальный, от Figma)

Ранний источник бьёт поздний:
1. Code Connect-снипеты → используй замапленный компонент кодовой базы напрямую
2. Ссылки на документацию компонентов → следуй им
3. Аннотации дизайнера → прямые указания
4. Дизайн-токены (CSS-переменные) → мапь на токен-систему проекта
5. Сырые hex / абсолютные координаты → дословно

## Обработка ошибок (официальные правила Figma)

- Ошибка `get_design_context` → СТОП, прочитай сообщение, потом ретрай.
- URL без `node-id` → спроси ссылку на ноду. Не гадать, не передавать пустой nodeId.
- Таймаут → повтори на ноде поменьше (секция/компонент вместо экрана).
- Запрещено молча падать в «сверстаю по скриншоту», пока design context доступен.

## Частые грабли

- **Retina-скриншоты 2x**: `get_screenshot` может вернуть картинку в 2х от размеров фрейма — сверяй пропорцию с width/height из metadata, pixel_diff.py сам приведёт размеры.
- **Урезанный design context у корня**: у экрана из 200 слоёв корневой ответ не содержит всех деталей — запрашивай контекст секций/компонентов отдельно.
- **Инстансы компонентов**: metadata показывает инстанс, а начинка живёт в main component. Если чего-то не хватает — запроси контекст самого инстанса по его node id.
- **Макет поменяли во время работы**: если context противоречит скриншоту, сними скриншот заново и скажи пользователю о расхождении.
